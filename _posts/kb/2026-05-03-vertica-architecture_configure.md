# 버티카 아키텍처

---

## Slide 1 — 버티카 기술요소

### 주요 특징

| 범주 | 내용 |
|------|------|
| **고가용성** | 마스터 노드 없음, 장애에 대한 안정성 |
| **확장성** | 유연한 확장성 보유 |
| **처리 방식** | Pure MPP |
| **저장 구조** | Columnar (순수 컬럼 기반 구조) |
| **압축** | Encoding / Compression |
| **성능** | Projections, 인덱스 없음, 분석 최적화 물리 구조 |
| **적재** | 병렬 적재 기능 보유 |
| **공간 효율** | 변경 작업 처리 공간 보유, 공간의 효율화, I/O 양의 감소 수행 |
| **알고리즘** | 컬럼별 차등 알고리즘 적용 |
| **관리** | 관리 편의성 |

## Slide 2 — 버티카 MPP

### MPP (Massively Parallel Processing) 분산 처리

- 모든 노드가 각자 CPU, 메모리, 디스크로 쿼리 병렬 처리
- 어느 노드이든 **initiator**, **executor** 역할 가능

```
Application
    └── Initiator (노드 1)
            ├── Executor (노드 2)
            └── Executor (노드 3)
```

### 분산 처리 구조에서 데이터 저장

#### 분산 Segment (기본)
- 기본적으로 한 테이블의 데이터는 **모든 노드에 분산**되어 저장

| 노드 1 | 노드 2 | 노드 3 |
|--------|--------|--------|
| 테이블 일부 | 테이블 일부 | 테이블 일부 |

#### 복제 Unsegment (선택)
- 필요에 따라 **작은 테이블은 모든 노드에 복제** 저장 지정 가능

| 노드 1 | 노드 2 | 노드 3 |
|--------|--------|--------|
| 테이블 전체 | 테이블 전체 | 테이블 전체 |

## Slide 3 — 클러스터 구성도

### 네트워크 구성

```
                  [ Public N/W ]
                  1G / 10G Switch
                       │
        ┌──────────────┼──────────────┐
        │              │              │
   NODE1 (Linux)  NODE2 (Linux)  NODE3 (Linux)
     Vertica        Vertica        Vertica
        │              │              │
        └──────────────┼──────────────┘
                  [ Private N/W ]
                    10G Switch
```

### 구성 설명

| # | 항목 | 설명 |
|---|------|------|
| **1** | Vertica 클러스터 | 3 node로 클러스터 구성 / 자체 이중화 (HA) |
| **2** | Public Network | 클라이언트 등 클러스터 외부와 통신하기 위한 서비스 전용 채널. 1Gb/10Gb로 구성 (고객사 백본망에 따라 상이할 수 있음) |
| **3** | Private Network | 버티카 클러스터 내부 통신을 위한 전용 채널. 10Gb로 구성 |
